## 2026-06-28 — CPU 性能分析 + 回退

### 做了什么
1. 完整代码级分析：`main.cpp` 结构、loop 执行顺序、非阻塞设计概览
2. 回答了代码是否适合 FreeRTOS → 否，单线程 `loop()` + `millis()` 是合理选择
3. 添加 CPU 监控（占用率 + loop 频率，每 5 秒串口输出）
4. 先后在 **80MHz** 和 **240MHz** 下实际 OTA 烧录测试 CPU 数据

### 测试数据
| 频率 | CPU 占用 | loop 频率 | 结论 |
|------|---------|-----------|------|
| 80MHz | ~94% | 1-2 Hz | AsyncWebServer 占大量 CPU |
| 240MHz | ~92% | 0-1 Hz | 瓶颈不是原始速度，是 WiFi stack 任务竞争 |

### 结论
- 瓶颈是 **AsyncWebServer + WiFi 后台 TCP 任务与 loop() 争用 ESP32 单核**
- 这不是 bug，这是 ESP32 Arduino 架构下正常现象
- loop 1Hz 不影响核心功能（时间显示靠 RTC 硬件，按键、NTP、天气、MQTT 均正常工作）
- **不需要优化**，不存在实际 bug
- 最终换回 80MHz 运行
